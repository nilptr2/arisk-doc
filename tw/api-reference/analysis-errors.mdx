### 響應結構

當 API 呼叫發生錯誤時，將返回包含以下欄位的 JSON 響應：

- `code`: 6 位數字的業務錯誤碼。
- `message`: 錯誤的具體描述資訊。

示例：
```json
{
  "code": "104001",
  "message": "Invalid request parameters (請求引數無效)"
}
```

### 錯誤碼格式

業務錯誤碼的第一位數字代表錯誤的級別或型別：

- **1**: 引數或輸入錯誤。
- **2**: 業務邏輯錯誤。
- **3**: 系統或內部錯誤。
